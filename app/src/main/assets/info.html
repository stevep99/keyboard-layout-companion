Keyboard Layout Companion is a tool for visualizing and generating images of alternative keyboard layouts and designs.

Modes

The app works in three different modes, selectable via the Mode dropdown:
- Display: for visualizing and editing keyboard layouts.
- Score: for calculating an "effort grid" for a keyboard for use in Keyboard Analyzer programs. The algorithm used for the calculation is defined in the Colemak-DH Model page.
- Distance: for calculating the travel distance for each key for its respective home finger, as measured using the algorithm described above.

Editing

The positions of keys can moved using drag-and-drop on the keyboard view. Tapping a key toggles the fade state. For more detailed editing, such as adding layers or adjusting key colours, a keyboard definition file must be copied to the device's SD Card, details below.

Keyboard Geometries

A number of common keyboard physical geometries are supported, including ANSI (common in North America), ISO (common in Europe), and JIS (common in Japan). Also supported are a selection of matrix-like keyboards. You can also create custom keyboard geometries.

The app also allows you to view additional variations of the most common keyboard geometries, including the Wide Mod and Angle Mod.

To add extra geometries: Create a geometry definition JSON file on the SD Card in the location:
/sdcard/keyboard/geometries

Layouts

Keyboard layouts are mappings between the physical key and the output that is produced when that key is typed. A wide variety of alternative layouts are included by default, but you can also add your own. Each layout can have one or more layers defined. If more than one layer is defined in a layout, controls will be shown making it possible to cycle through the available layers. It also possible to configure the default key background colour.

To add extra layouts create a layout definition text file on the SD Card in the location:
/sdcard/keyboard/layouts

Other Options

The "Filter" dropdown allows only a chosen portion of the keyboard to be rendered.

The "split" checkbox will render a version of the keyboard with left- and right-hand sides split. The spacebar will also be split into two in this case.

The "styles" checkbox allows any extra styling on the keys to be shown, such as home key indicators or fading.

The "fingers" checkbox causes the displayed keyboard to be colour-coded by finger used for each key. When deselected, the colours defined in the layout definition file will used if defined, or otherwise, by key type.

The "Settings" button on the titlebar allows keyboard images to be generated using either of two types key graphics: a "rounded" style, designed by SteveP, and used on the Colemak-DH site; or a "square" style designed by DreymaR and used in EPKL.

The Title Bar contains buttons to print the current keyboard to an image file, and to write the current layout as a text file. The files are saved on the SDCard in the location:
/sdcard/keyboard/output
